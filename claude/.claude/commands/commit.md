Commit staged changes (or stage and commit all modified tracked files if nothing is staged).

$ARGUMENTS

Steps:
1. Run `git diff --staged` and `git status` to understand what's changing
2. Run `git log --oneline -10` to learn this project's commit message style
3. Stage files if needed
4. Write a commit message that matches the style from step 2 - same tense, capitalisation, length, and format. Match the description/body style too: if existing commits have no body, do not add one. Do not add details the existing style does not include.

Do not add a Co-Authored-By line.
